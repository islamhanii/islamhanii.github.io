BLOCK & STREAM CIPHER

Introduction

The internet of things rapidly growing day over day and the world work to secure it more and more. Secure file transfer protocols such as FTPS, SFTP/SCP, HTTPS and WebDAVS encrypt and decrypt data through symmetric key ciphers (private key). Today we will discuss all about these ciphers which classified into two main groups: Stream ciphers and Block ciphers.

Block Cipher

An encryption algorithm that encrypts and decrypts a fixed size of n-bits of data(Block) at time. The usual size of each block are 64 bits or more than 64 bits. It takes a 64 bits of plain text and encrypt it to 64 bits of cipher-text. In case if the plain-text is small than 64 bits block, we use the padding schemes. The popular block ciphers used today are DES, 3DES, AES, IDEA, Twofish, and Blowfish.

Stream Cipher

An encryption algorithm that encrypts and decrypts 1 bit or byte of plaintext at time. It uses an infinite stream of pseudorandom bits as the key. This pseudorandom should be unpredictable and never reuse the key. Its key should be longer than the plain-text as example if you want to encrypt 500 Mb video file, you need at least 4 Gb key. The popular stream ciphers used are SALSA, SOSEMANUK, PANAMA, and many others, but not have the popularity as RC4.
